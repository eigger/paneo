# Paneo 설계 문서

> 프로젝트명: **Paneo** (작업 저장소 경로는 `easydash`, 제품/패키지명은 Paneo).
> 위젯 `pluginId` 네임스페이스·컨테이너 이미지·UI 브랜딩은 Paneo 기준.

> 라즈베리 파이·상시 디스플레이 장치를 위한 대시보드 플랫폼.
> **웹에서 드래그&드롭으로 편집 → 디스플레이에 리로드 없이 실시간 반영.**

- 상태: Draft v0.1
- 작성일: 2026-07-03
- 대상 규모: 소수 다중 기기 (집/사무실 방마다 화면)
- 위젯: 플러그인 생태계 (서드파티 확장 가능)
- 호스팅: **셀프호스트 확정** — 클라우드 아님. 디스플레이와 같은 기기 병설(co-locate) 허용 (§10)

## 1. 목표 (Goals)

1. **사용자 친화적 편집**: 웹 에디터에서 드래그&드롭으로 그리드 기반 대시보드 레이아웃 구성.
2. **초안/발행 모델**: 편집은 자유롭게 수행하고 "적용(Publish)" 버튼 클릭 시 기기에 실시간 동기화.
3. **경량 키오스크**: 디스플레이 기기(라즈베리 파이 등)는 Chromium 키오스크 모드로 구동하는 웹 뷰어 환경 제공.
4. **다중 기기 및 그룹 관리**: 기기별/그룹별 레이아웃 지정 및 일괄 배포 지원.
5. **플러그인 구조**: 매니페스트 기반의 독립된 서드파티 및 내장 위젯 확장 구조 제공.
6. **성능 프로파일 지원**: 기기 사양(High/Low 성능)에 맞춘 애니메이션 제한 및 기능 저하(degrade) 지원.
7. **오프라인 회복력**: 네트워크 단절 시 서비스 워커 캐싱을 통해 마지막 수신 화면 유지.

## 2. 비목표 (Non-Goals)

- 범용 BI/모니터링 대시보드 아님 (상시 정보 표시용 엠비언트 디스플레이 타깃).
- 고성능 터치/게임 인터랙션 및 결제/테넌트 관리 등의 고복잡도 기능 제외.
- 멀티테넌트 SaaS 구조 제외 (단일 환경/자체 호스팅 기준).

## 3. 사용자 & 시나리오

| 역할 | 정의 및 기능 |

| **편집자 (Editor)** | 웹 브라우저를 통해 기기 관리, 레이아웃 편집, 실시간 프리뷰 및 적용 수행 |
| **디스플레이 (Display)** | Pi 키오스크 런타임. 수신한 발행 레이아웃을 렌더링하고 폴링 데이터 갱신 |

- **시나리오**: 편집자가 웹 에디터에서 레이아웃 수정 후 적용 버튼 클릭 시, 해당 기기에 할당된 화면이 실시간으로 동기화됨.

### 4.1 컴포넌트

- **서버 (허브)**: 레이아웃/기기 메타데이터 SQLite 영속성 관리, 외부 API 캐싱/CORS 회피용 데이터 프록시, WebSocket 기반 상태 전송.
- **웹 편집기**: React 기반 SPA. 그리드 배치 및 속성 폼 조작, 기기별 프리뷰 표시.
- **디스플레이 클라이언트**: Chromium 키오스크 환경. WebSocket 연동을 통해 레이아웃을 전송받으며 오프라인 캐싱 지원.
- **컴패니언 에이전트 (선택)**: 라즈베리 파이 등 호스트 OS 수준의 제어(화면 전원, 시스템 업데이트, 워치독)를 수행하는 백그라운드 서비스 데몬.

### 4.2 구조 설계 근거

- 다중 기기 관리를 위해 단일 허브 서버 구조 선택.
- 디바이스의 프로비저닝을 단순화하기 위해 디스플레이는 웹 페이지 뷰어 형태로 설계.
- API 키 은닉 및 캐싱 제어를 위해 외부 요청은 서버 프록시로 일원화.

### 4.3 성능 프로파일

| 항목 | High 티어 (Pi 4/5, PC) | Low 티어 (Pi 3, Zero 2) |
|---|---|---|
| 디스플레이 런타임 | 풀 기능 React 렌더링 | 무거운 모듈 로딩 생략, 경량 구동 |
| 애니메이션 | 페이드, 슬라이드, Ken Burns 활성 | 애니메이션 비활성화 (정적 전환) |
| 미디어 재생 | HTML5 비디오, WebRTC 재생 허용 | 비디오 비활성 또는 정적 스냅샷 대체 |
| 데이터 폴링 | 짧은 주기 갱신 | 주기 완화로 발열 및 CPU 부하 제어 |

### 4.4 국제화 (i18n) & 로케일

- **편집기 UI 언어**: 에디터 접속 계정/브라우저 로케일 기준 UI 텍스트 다국어화 (ko/en 등).
- **디스플레이 로케일**: 기기 속성(`locale`, `timezone`)에 기반해 위젯 내의 날짜/시간/숫자 형식을 `Intl` 포맷터로 개별 렌더링.
- **번들링**: 리눅스 디바이스의 다국어/이모지 렌더링 호환을 위해 로컬 Pretendard 폰트 서빙 및 Noto Color Emoji 의존성 기본 포함.

## 5. 데이터 모델

```
Device        기기 정보. { id, name, token, resolutionW, resolutionH, performanceProfile, locale, timezone, powerSchedule, agentPresent }
DeviceGroup   방 단위 기기 그룹. { id, name }
Layout        화면 구성 정보. { id, name, grid{cols,rows,gap}, background, draft{}, published{}, publishedAt }
WidgetInstance배치된 위젯 인스턴스. { id, pluginId, x, y, w, h, config{}, customCss }
Plugin        위젯 플러그인 정보. { id, version, type: "module"|"iframe", entry|url, manifest }
```

## 6. 동기화 모델

편집기 조작 중 발생하는 수정 사항은 `draft`에 임시 저장(실기기 미영향)되며, "적용(Publish)" 호출 시 `published`로 복사된 후 기기에 전송됨.

```
1) 에디터 편집 ──> draft 업데이트 (서버 자동저장)
2) 에디터 "적용" 클릭 ──> 서버: draft를 published로 복사
3) 서버 ──> 디바이스 WS: { type: "layout.set", layout: published } 형태로 전송 및 갱신
```

- **데이터 갱신**: 위젯 데이터는 프록시 폴링 주기에 맞춰 독립적으로 브로드캐스트(`widget.data`).

## 7. 플러그인/위젯 시스템

### 7.1 매니페스트 규격

```json
{
  "id": "weather",
  "name": "날씨",
  "version": "1.0.0",
  "defaultSize": { "w": 3, "h": 2 },
  "minSize": { "w": 2, "h": 2 },
  "configSchema": {
    "location": { "type": "string", "label": "지역", "required": true },
    "units": { "type": "enum", "options": ["metric","imperial"], "default": "metric" }
  },
  "data": {
    "source": "openweather",
    "pollInterval": 600
  }
}
```

### 7.2 렌더러 인터페이스

위젯은 표준 렌더 함수 구현체로 빌드됨:
```javascript
render(el, config, ctx) => void
```
- `el`: 위젯 컨테이너 DOM 엘리먼트.
- `config`: 스키마에 정의되어 영속화된 위젯 속성 객체.
- `ctx`: 로케일, 타임존, 성능 프로파일, 렌더 속성 등을 포함한 런타임 콘텍스트.

### 7.3 샌드박싱 및 플러그인 타입

1. **내장 위젯 (In-tree)**: `/shared/widgets.js`에 내장된 코어 위젯 그룹.
2. **모듈 플러그인 (Module)**: `data/plugins/` 디렉토리에 직접 수동 배배포되어 서버 실행 시점에 병합 및 실행되는 관리자 신뢰 컴포넌트.
3. **아이프레임 플러그인 (Iframe)**: 외부 URL을 지정하고 iframe sandbox 옵션(`strict`, `scripts`, `trusted`) 하에 격리되어 실행되는 격리형 위젯.

## 8. 확장 및 연동 아키텍처

- **RTSP 카메라**: 브라우저의 RTSP 디코딩 제약 우회를 위해 서버 단에 `go2rtc`/`MediaMTX`를 두고 WebRTC/HLS/Snapshot 형태로 변환 후 위젯에 매핑.
- **이미지 액자**: `paneo.photo` 미디어 슬라이드쇼를 확장하여 로컬 디렉토리, Immich 등의 소스를 활용하고 스케줄과 결합해 활용.
- **Home Assistant 연동**: 서버 사이드에서 HA 토큰을 은닉 및 관리하고 WebSocket/REST API 프록싱을 통해 센서/스위치/날씨 카드를 구성.
- **ESP32 디스플레이**: 저사양 TFT/E-Ink 기기를 위해 서버가 Puppeteer 등으로 레이아웃을 스크린샷 캡처(BMP/PNG)하여 ESP32에 푸시하는 렌더링 모델 지원.

## 9. 화면 전원 제어

디렉션 전원 관리 및 번인 대응용 OS 제어:
- **실행기**: 컴패니언 에이전트가 호스트에서 Wayland(`wlr-randr`), X11(`xset`), 또는 레거시 `vcgencmd`를 통해 물리 화면을 제어.
- **트리거**: 24시간제 텍스트 기반 전원 스케줄(`powerSchedule`)에 맞춰 작동하거나, 모션 센서 및 원격 수동 API 호출에 의해 실행.

## 10. 호스팅 및 배포

- **기본 호스팅**: 셀프 호스팅을 기본으로 하며 SQLite 및 사진 데이터를 named volume으로 영속하는 단일 Docker 컨테이너 구조 제공.
- **기기 배치**: 성능이 검증된 단일 Pi 4+에 서버와 디스플레이 브라우저를 병설하여 운영하거나, NAS/홈 서버에 서버 컨테이너를 단독 구성하고 디바이스는 씬 클라이언트로 동작하도록 선택적 배치 지원.

## 11. 기술 스택

- **서버**: Node.js + Fastify + `ws` + SQLite (`node:sqlite` 내장 모듈 사용으로 컴파일 의존성 없음).
- **웹 편집기 / 디스플레이**: Vanilla JS + CSS Grid + `react-grid-layout` 호환 캔버스 레이아웃 엔진.
- **프로비저닝**: Docker + systemd 서비스를 활용한 원클릭 셸 스크립트 기반 온보딩 및 원격 업데이트 자동화.

## 12. 보안 및 격리

- **디스플레이 연결**: 기기 페어링 시 발급되는 URL 토큰 기반 단순 인증 적용 (읽기 전용 캔버스 수신).
- **편집기 인증**: scrypt 해시 기반 관리자 단일 암호 세션 로그인 필수화, HttpOnly + SameSite=Lax 쿠키 세션 수립.
- **자동화 API 토큰 (D69)**: 세션 쿠키를 가질 수 없는 비-브라우저 클라이언트(예: Home Assistant `rest_command`)용 `Authorization: Bearer <token>` 인증. 세션과 동일 권한이며 에디터 설정 패널에서 조회/재발급.
- **XSS 방지**: iCal, RSS 등의 외부 데이터 주입 시 `escapeHtml` 및 `safeHttpUrl`을 공통 적용하여 보안 위협 격리.
